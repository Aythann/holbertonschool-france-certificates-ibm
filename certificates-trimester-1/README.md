# IBM Certificates – Trimester 1  
## Technical & Professional Certifications Documentation

---

## 1. Introduction

This document presents the **IBM certification programs** completed during **Trimester 1** as part of the **Holberton x IBM partnership**.

These certifications aim to validate **foundational technical knowledge** and **professional methodologies** required for software engineering roles.

They are mandatory components of the first-year curriculum and must be completed and validated according to Holberton guidelines.

---

## 2. Certification Programs Overview

Two certification programs are included in Trimester 1:

- **Introduction to Open Source**
- **Agile Explorer – Powered by Agile at IBM**

Both courses are introductory and require **no prior knowledge**.

---

## 3. Certificate 1 – Introduction to Open Source

### 3.1 General Information

- **Platform**: IBM SkillsBuild / SkillsNetwork  
- **Estimated Duration**: ~4 hours  
- **Level**: Beginner  

Course link:  
https://skillsbuild.skillsnetwork.site/courses/course-v1:IBM+OS0101EN+v1

---

### 3.2 Learning Objectives

This certification covers:

- Open Source Software (OSS) fundamentals
- Choosing an open source project
- Contributing to open source projects
- Collaboration tools
- GitHub basics
- Best practices for contributions
- Licensing and governance models

---

## 4. Certificate 2 – Agile Explorer (Powered by Agile at IBM)

### 4.1 General Information

- **Platform**: IBM SkillsBuild  
- **Estimated Duration**: ~1 full day

## 5. Repository Structure

The certificates must be stored following the required structure:

- **Repository**: `holbertonschool-france-certificates-ibm`
- **Directory**: `certificates-trimester-1`
- **Files**:
  - `certificate-oss` (image format)
  - `certificate-agile` (image format)

---

## 6. Validation Process

Once a certificate is obtained:

1. Upload the certificate image to the correct directory
2. Respect the required file naming conventions
3. Share the certificate link if available
4. Request verification from the local **SWE**
5. The project will be manually reviewed and validated
